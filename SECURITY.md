# Security Policy

## Reporting a security issue

If you identify a security issue in this repository, please do not open a public issue with sensitive details.

Instead, report the issue privately to the repository owner through GitHub or another trusted contact channel.

## Scope

This repository is primarily a documentation and knowledge-base project. Security reports may include:

1. Exposed sensitive information.
2. Unsafe technical guidance.
3. Broken links to malicious or compromised resources.
4. Incorrect security claims that could create operational risk.
5. Content that unintentionally enables harmful misuse.

## Handling principles

Reports will be reviewed according to the following principles:

1. Defensive purpose.
2. Responsible disclosure.
3. Minimization of operational risk.
4. Correction of inaccurate or unsafe material.
5. Preservation of academic and professional integrity.
