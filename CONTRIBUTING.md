# Contributing

Thank you for considering a contribution to Cyber Attribution.

## Contribution goals

Contributions should improve the repository as a defensive, educational, and research-oriented cyber attribution knowledge base.

Useful contributions include:

1. Cyber attribution frameworks.
2. AI-based attribution methods.
3. False flag analysis methods.
4. Incident response and evidence preservation frameworks.
5. Detection engineering resources.
6. Structured analytic techniques.
7. Legal and policy attribution references.
8. APA 7 references.
9. Corrections to inaccurate or outdated material.

## Quality expectations

Contributions should be:

1. Clearly written.
2. Defensive in purpose.
3. Properly sourced.
4. Structured in Markdown.
5. Consistent with APA 7 citation style where references are used.
6. Respectful of copyright and licensing constraints.

## Pull request checklist

Before submitting a pull request, check that:

1. The contribution has a clear purpose.
2. Sources are reliable and cited.
3. Evidence is separated from assessment.
4. Confidence, uncertainty, and assumptions are stated where relevant.
5. Tables are readable in GitHub Markdown.
6. Links are working.
7. No sensitive or private information is included.
